I AM | 바이브코딩을 활용한 개인 PJT

안녕하세요, SSAFY 16기 정지혁입니다

다양한 아이디어를 실제 서비스로 만드는 것을 좋아합니다.

SSAFY 에서 다양한 프로젝트를 진행하며 함께 성장하려고 합니다.

자격증 시험 학습 서비스의 개선 포인트와 개인화 서비스를 합친 임고봇이라는 웹사이트를 운영하고 있습니다.

총 2,000+명의 회원수를 기록하였고, 관련 도메인에서 검색노출 상위권을 차지하고 있습니다.

기술 스택

웹 표준을 기반으로 한 견고한 구현을 우선합니다. 특정 도구에 화면 구조를 종속시키기보다 HTML의 의미, CSS의 레이아웃 원리, JavaScript의 실행 모델을 이해하고 프로젝트의 규모와 팀 상황에 맞는 기술을 선택합니다.

  • Next.js 서버 사이드 랜더링 기술을 통한 SEO 최적화, 화면 로딩 속도 개선등의 효과를 보기 위해 채택하였습니다.
  • RN / Expo iOS, Android 모바일 앱 개발 시간 단축 효과를 위해 채택하였습니다.
  • MySQL 풍부한 레퍼런스, 높은 안정성, 쉬운 학습 곡선을 가지고 있어 채택하였습니다.
  • AWS ECS , RDS , Elastic Beanstalk, cognito , cloudfront, SES , S3 등 배포부터 유지보수까지 클라우드 인프라를 관리할 수 있습니다.

프로젝트

저는 단순 구현으로만 끝나지 않는 지속 가능한 프로젝트를 만들고자 노력하고 있습니다.
다음 두 프로젝트는 제가 진행했던 프로젝트 중 가장 반응이 좋았던 프로젝트입니다.

임고봇

총 사용자 수 2024명이 사용중인 초등·중등·특수 임용고시 시험 대비 학습 사이트입니다.
Upstage Document Parsing API 를 활용한 문서 구조화 데이터 파싱 및 퀴즈 출제 자동화,Next.js, React Native , EXPO 를 활용하여 웹 / iOS,Android앱을 동시 운영중입니다.

리뷰인코리아

한국에서의 일상을 글로벌 사용자들에게 소개하는 블로그형 웹사이트입니다.
Next.js 와 i18n , OpenAI API 를 사용하여 17개 언어 번역 자동화를 구현하였습니다.

문제 해결 방식

문제를 발견하면 바로 코드를 바꾸기보다 먼저 재현 조건과 영향을 받는 범위를 정리합니다. 네트워크 요청, 렌더링 시간, 브라우저 이벤트와 사용자 행동을 분리해 관찰하고, 개선 전후를 같은 조건에서 비교할 수 있는 기준을 만듭니다.

성능 문제는 Lighthouse 점수 하나로 판단하지 않습니다. 실제 화면에서 중요한 콘텐츠가 언제 보이는지, 입력에 얼마나 빠르게 반응하는지, 불필요한 JavaScript가 사용자 기기에서 어떤 부담을 만드는지를 함께 확인합니다. 해결책 역시 라이브러리 추가보다 데이터 구조 단순화, 렌더링 범위 축소, 캐시 정책 정리처럼 원인에 가까운 변경을 우선합니다.

예측 가능한 실패를 설계합니다

좋은 화면은 성공한 상황만 자연스러운 화면이 아닙니다. 로딩, 빈 결과, 권한 부족, 네트워크 단절과 서버 오류를 별도의 예외가 아니라 제품 흐름의 일부로 취급합니다. 사용자가 현재 상태와 다음 행동을 이해할 수 있도록 메시지와 복구 수단을 함께 제공합니다.

협업과 품질

팀이 빠르게 움직이려면 개인의 구현 속도보다 변경을 이해하고 검증하기 쉬운 구조가 중요하다고 생각합니다. 작업을 작은 단위로 나누고, Pull Request에는 변경 이유와 확인 방법, 남아 있는 위험을 함께 기록합니다. 리뷰에서는 취향보다 사용자 영향, 일관성, 복잡도와 운영 비용을 기준으로 의견을 나눕니다.

  • 접근성 의미 있는 마크업, 명확한 포커스, 키보드 조작과 충분한 대비를 기본 구현 조건으로 둡니다.
  • 테스트 구현 세부사항보다 사용자가 실제로 수행하는 핵심 흐름과 실패 복구 과정을 검증합니다.
  • 문서화 결정의 배경과 제약을 남겨 다음 작업자가 같은 문제를 처음부터 다시 풀지 않도록 합니다.
  • 관찰 배포 이후 오류와 성능 지표를 확인하고, 실제 사용 패턴을 다음 개선의 근거로 연결합니다.

앞으로

홍길동은 프론트엔드를 화면 구현에 한정하지 않고 제품과 시스템 사이를 연결하는 역할로 바라봅니다. 브라우저 렌더링, 네트워크, 서버 사이드 렌더링과 웹 성능에 대한 이해를 계속 넓히며, 사용자가 체감하는 품질과 팀이 유지할 수 있는 구조를 함께 개선하고자 합니다.

새로운 기술을 빠르게 도입하는 것보다 문제에 적합한 기술을 선택하고 그 선택을 설명할 수 있는 개발자를 지향합니다. 작고 명확한 개선을 반복하며 오래 사용할 수 있는 제품을 만드는 것이 홍길동의 다음 목표입니다.
